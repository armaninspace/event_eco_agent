## Purpose

This file is a simple consolidated economic table for analysis, review, and
handoff. It combines weekly city and county Womply economic observations with
available city or county attributes and adds an `msa` column.

County rows use county FIPS as the bridge to CBSA/MSA geography. City rows are
included in the same table but are not assigned an MSA because the city source
does not provide a county bridge, and MSAs/CBSAs are county-based geographies.

## MSA Mapping Logic

MSAs/CBSAs are county-based geographies. Following `docs/zip_to_msa.doc`, the
defensible mapping path is:

1. Start from county FIPS.
2. Join county FIPS to the official county-CBSA crosswalk.
3. Use the matched CBSA name as `msa`.

For this dataset:

- County rows use `countyfips_str` joined to `county_fips` in
  `data/processed/us/county_cbsa_crosswalk_us.csv`.
- City rows are marked `not_mapped` because a city can cross county boundaries
  and the source city table does not provide a county allocation.
- Unmatched county rows are retained with blank `msa` so the table does not drop
  non-metropolitan or otherwise unmatched county observations.

## Known Limitations

- This is a simple wide table, not a normalized dimensional model.
- City rows do not receive MSA values because there is no county bridge in the
  city economic source.
- County rows without a CBSA crosswalk match keep blank `msa`; this can include
  counties outside metropolitan/micropolitan areas.
- The file includes only city and county Womply weekly data, not state or
  national economic rows.
- Economic measures are observational activity measures and are not causal
  estimates of sports-event effects.

## Suggested Downstream Use

Use this file for quick economic coverage checks, city/county weekly
exploration, and joins to event exposure by `week_start_monday`. For strict
MSA/CBSA analysis, filter to `geography_level == "county"` and
`msa_mapping_status == "matched"`, or use existing CBSA-level modeling outputs
that aggregate county data with population weights.
