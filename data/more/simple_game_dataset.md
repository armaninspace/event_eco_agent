## Purpose

This file is a simple consolidated game-level table for analysis, review, and
handoff. It keeps all columns from the enriched sports game table and adds
game-level MSA/CBSA fields where exact U.S. venue geography is available.

The `docs/zip_to_msa.doc` column points each row to the local documentation note
that explains how ZIP or address data should be mapped to MSA/CBSA geographies.

## Geography Notes

MSAs/CBSAs are county-based geographies. `docs/zip_to_msa.doc` explains the
bridge required for ZIP or address data:

1. Convert ZIP or address to county FIPS.
2. Join county FIPS to an official Census/OMB CBSA or MSA delineation table.
3. Use that county-based result as the MSA/CBSA geography.

Important caveat: ZIP codes can cross county boundaries. A ZIP-to-MSA mapping
can therefore be one-to-many unless a county allocation rule or address-level
geocoder is used.

In this file, `msa` is populated by joining each game to
`data/processed/us/sports_games_us.csv` on `game_id`. That U.S. sports file is
built from exact venue geography, where county FIPS is joined to a Census/OMB
CBSA crosswalk. Games without an exact U.S. venue match are retained with blank
MSA fields.

## Known Limitations

- The table is intentionally wide and simple; it is not normalized into separate
  game, team, venue, and geography tables.
- MSA is populated only where exact U.S. venue geography is available.
- Some Canadian, non-U.S., neutral-site, or unmatched games retain blank MSA
  fields.
- Source season type values are not harmonized across leagues.
- Future games may appear for sources that include scheduled games.
- Attendance availability varies by league and source.
- The ZIP/MSA documentation column is provenance; the mapped value is `msa`.
